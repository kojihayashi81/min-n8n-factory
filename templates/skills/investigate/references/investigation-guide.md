# 調査ガイド

## 調査の目的

Issue の背景・要因・影響範囲を明らかにし、次フェーズ（仕様化）の判断材料を揃えること。
実装の是非や具体的な解決策はこのフェーズでは決めない。

## 調査の進め方

### 1. Issue を読み込む

- タイトルと本文から「何が問題か」「何を実現したいか」を把握する
- ラベル・マイルストーン・リンクされた PR を確認する

### 2. コードベースを調べる

- 関連するファイル・関数・型を `Grep` / `Glob` で特定する
- 既存の実装パターンを把握する（後の仕様化で一貫性を保つため）

### 3. 既存のノート・仕様を確認する

- `openspec/investigations/` に類似 Issue の調査ノートがあれば参照する
- `openspec/specs/` に関連仕様書があれば参照する

### 4. 不確かな点を明示する

調査で解消できなかった点は「要確認事項」に残す。
憶測で埋めない。

## 調査ノートのフォーマット

[調査ノートテンプレート](investigation-note-template.md) を参照してください。

## やってはいけないこと

- 実装コードをこのフェーズで書く
- 要確認事項を憶測で埋める
- 調査ノートを 1 ファイルにまとめずに分散させる
