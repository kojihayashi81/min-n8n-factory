# Slack 通知

ワークフローの成功・失敗を Slack に通知する。通知先は1チャンネルに集約し、Issue 単位でスレッドにまとめる。将来的にワークフロー動作ログのスタックとして活用する。

## 方針

- Bot Token 方式で認証する
- Block Kit ボタンで GitHub Issue / PR / n8n 実行ログへ遷移できるようにする
- 調査フローだけでなく、将来のレビュー・承認フローにも対応可能な設計にする

## メッセージ構造

Issue 単位でスレッドにまとめる。処理開始時に親メッセージを投稿し、完了・失敗はスレッド返信する。

```text
🔄 処理開始: #42 ログイン画面のエラーハンドリング改善   ← 親メッセージ
  └─ ✅ 調査完了  ⏱️ 3分12秒                         ← スレッド返信
     [ 📋 Issue #42 ]  [ 🔀 PR #43 ]
```

詳細は [message-format.md](./message-format.md) を参照。
