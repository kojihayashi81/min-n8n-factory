## リソーステンプレート

設計時に検討した動的リソーステンプレート。allowlist（`mcp-server/src/lib/allowlist.ts`）は定義済みだが、ルートは未登録。

| テンプレート | 用途 |
| --- | --- |
| `project://file/{path}` | 公開許可されたファイルの原文参照 |
| `project://workflow/{name}` | 特定ワークフローの整形済み説明 |
| `project://command/{name}` | 特定コマンドの説明 |

`project://file/{path}` は allowlist に含まれるパスのみに制限する前提。

## ファイル Watcher

現在はコンテナ起動時に一度だけパイプラインを実行し、結果を in-memory キャッシュしている。公開対象ファイルの更新時刻が変わったときに自動再生成する仕組みは未実装。

導入する場合の方針:

- ファイル更新時刻の比較で変更検知
- 変更があったノードだけ再計算（DAG ノード単位のキャッシュ）
- 当面はコンテナ再起動で対応可能

## 改善候補

### 検索の表記ゆれ対応

現在の検索は単純な部分文字列マッチ。同義語や表記ゆれ（例:「setup」と「セットアップ」）には対応していない。

### ラベル定義の動的抽出

`build-labels-lifecycle.ts` の `SPEC_LABELS` がハードコードされている。`docs/setup.md` からラベル定義を動的に抽出すれば、仕様変更時にコード修正が不要になる。

### ドリフト検出ルールの拡充

現在の検出対象はラベル定義、ワークフロー TODO、make ターゲットの 3 領域。以下の追加を検討:

- セットアップ手順とスクリプトの整合性
- テンプレート配布物の整合性

### spec ブースト率の調整

検索時の spec ブースト（現在 1.5 倍）が適切かは、実際のクエリパターンで検証が必要。[精度評価ガイド](./evaluation-guide.md) の Layer 2 テストケースで確認できる。
